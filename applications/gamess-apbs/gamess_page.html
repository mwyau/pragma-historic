INSTALLATION

For Linux i386 or ia64 clusters

  1. Download the corresponding RPM file (above)
  2. Install the RPM file on all nodes in your cluster
    The default installation path is /opt/gamess. If you want to install to a different path, specify it using RPM options.
  3. Edit $GMSPATH/bin/rungms and $GMSPATH/machines/rungms
    Set the scratch directory for users in the line
    set SCR=/state/partition1/$USER
  4. Inform users the installation path
  5. Instruct users to create their own scratch directories in the correct path.

For Solaris clusters

(Substitute $GMSPATH with the path of gamess installation directory)
  1. Edit the $GMSPATH/gamess file
    1. change the line "default=/work/jpg/gamess" (line 7) to $GMSPATH
  2. Edit the $GMSPATH/rungms file
    1. change the line "setenv ERICFMT /work/jpg/gamess/ericfmt.dat" (line 107) to $GMSPATH
    2. change the line "setenv MCPPATH /work/jpg/gamess/mcpdata" (line 108) to $GMSPATH
    3. change the line " if ($os == SunOS) set GMSPATH=/work/jpg/gamess" (line 287) to $GMSPATH. Its also useful to have GMSPATH set in your environment.

VERIFICATION

You can run gamess either interactively or through a local job scheduler, such as SGE, PBS, LSF, SQMS etc.

Before you run gamess for the first time, you need to create a srach directory on all nodes. For example, on a Rocks cluster, you can run

	cluster-fork mkdir /state/partition1/$USER

To run interactively on a single processor (on linux), execute:
/opt/gamess/bin/rungms file_prefix 00 1 2>&1 > file_prefix.out

To run interactively (on SUN), execute:
$GMSPATH/rungms file_prefix 00 1 >& file_prefix.out

where "file_prefix" is the file prefix of the gamess input file "gamess_prefix.inp"

To run through the queue, run the script "gamess" (/opt/gamess/bin/gamess on linux or $GMSPATH/gamess on sun's).
At the prompt, type:

gamess -n PROCS file_prefix

where PROCS is the number of gamess processes you want.
