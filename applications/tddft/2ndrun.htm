Summary Report

2nd User Run

 

This is a summary report of my experiences as an application user in running TDDFT as part of our first routine-basis experiment.

 

Runs are initiated from my user account on ume at AIST Japan.

First run started on 8/3/04.

 

As of 08/27/04 4pm PST:

 Total number of runs: 8 (excluding short tests and user terminated runs)

 Longest run: 97 hours

Shortest run: 12 hours

Average run: 38 hours

Number of terminations due to hangs: 2

Number of terminations due to faults: 5

Probable causes for application terminations or hangs: (no concrete prove, only correlation)

1.      Server or compute nodes down

2.      Failure to recover from fault

3.      User environment setup change

4.      PE configuration change

 

Steps required to add a server to the run:

  1. contact the server administrator to setup a user account, require site CA files, user SSH public key and DN.
  2. test SSH and GLOBUS access to the server
  3. test TDDFT run

Problems encountered:

  1. Some systems require passwordless ssh to run on compute nodes
  2. Problem with PE allocation scheme $fill_up on some systems due to application high memory demand
  3. Lack of memory on compute nodes
  4. Run-time libraries paths not set
  5. Bad CA files
  6. Different DN format requirements between sites
  7. Different software requirements between sites (gsissh)
  8. Problem in determining causes of errors or of application terminations
