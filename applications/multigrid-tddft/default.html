  • Account application
    1. The user had have a PRAGMA account before the Multi-Grid experiment.
    2. The user requested a TeraGrid account on the Web.
    3. The account was creaated on only UC/ANL site for the demo without the formal process.

  • Deployment
    1. The user wrote a system requirement for the application.
    2. The GOC sent the requirement to the PRAGMA sites and the UC/ANL.
    3. After notification, the user confirmed the requirement is met on the site.
    4. The user upload an input data file (212 MB) to each site.
    5. The user upload an application program written in Fortran 90.
    6. The user compiled the program on the site.

  • Test on each site
    1. Application level test
      • The user checked if the program works normally on a signle node.
    2. Local scheduler level test
      • The user tested a simple job submission to a local scheduler.
    3. Globus level test
      • The user tested a simple GRAM job submission from the client node to the cluster.
    4. Ninf-G level test
      • The user tested the TDDFT/Ninf-G/Globus application using the client node on AIST and a single node on the site.
