Information Sharing
  • Problems
    • Lack of info, documentation
    • Different time zones, admin domains, procedures
  • What we have done
    • Grid Operation Center: users guides, system info, status http://pragma-goc.rocksclusters.org
    • Site technical contacts, mailing lists, VTCs
  • What we plan to do
    • Create/implement/improve procedures, policies for production grid
10
Trust and Access
  • Problems
    • Different requirements, policies and software
  • What we have done
    • Site: trust each others CA
    • Do whatever is required to allow necessary access for PRAGMA projects
    • User: CA files, DN, SSH public key
  • What we plan to do
    • Setup PRAGMA CA
    • Automate CA management tasks
    • Automate users management tasks
11
Resource Requirements
  • Problems: unforeseen requirement details
  • What we are doing
    • Develop resource requirements thru running popular applications
      • Hardware (cpu, memory, disk space, network)
      • System configuration (public ip)
      • Software (versions, flavors, front-end/compute-notes)
    • Check resource requirements
      • NCSA grid status test, every 4 hours
      • http://rocks-52.sdsc.edu/pragma-grid-status.html
  • What we plan to do
      • INCA http://tech.teragrid.org/inca/www/documentation.html
12
User/Application Environment
  • Problems
    • Setup/test on each system
    • Different security policies/setups, software, version, path 
  • What we plan to do
    • Provide common user environment setup files
    • Automate setup/replication
      • Gfarm, http://datafarm.apgrid.org/testbed
13
Job Submission
  • Problems:
    • Resource selection
    • Test/implement for each system
    • Different schedulers, queues/PEs 
  • What we have done
    • Find common requirements thru running applications
  • What we plan to do
    • Easy user interface and automation for resource selection and job submission: GridPort, Grid Service
    • Middleware wrap some resource differences
14
Resource/Job Monitoring
  • Problems:
    • Different monitoring software don't work togather for centralized monitoring (SCMSWeb, Ganglia)
    • No centralized job monitoring
    • No job status change notification
  • What we are doing:
    • Standardize with SCMSWeb (Kasetsart University)
    • http://pragma-goc.rocksclusters.org/scmsweb/scms_home.html
  • What we plan to do:
    • Add SGE, PBS and LSF support in SCMSWeb
15
Resource/Job Accounting
  • Problems:
    • Usage data is at each system/node
    • No Grid-enabled accounting
  • What we have done:
    • Middleware collect some data
  • What we plan to do:
    • GridBank, http://www.gridbus.org

Fault Tolerance
  • Problems:
    • Many possible causes for faults
    • Crash jobs
    • Leave defunc processes and jobs
  • What we have done:

      • Better error reporting
      • Fault detection and recovery
    • Easy clean-up utility (GXP)
  • What we plan to do:
    • More improvements and solutions thru middleware
      • Dynamically add/remove sites
