Routine-Basis Experiments
  • Daily runs of TDDFT application on 10 sites for 3 months (July 1 ~ August 31)
  • Implement TDDFT with Ninf-G2 (It is a GridRPC application)
  • Figure out our testbed status and issues for real science
3
Experiment Procedure
  • Application of user account
    • We contacted each technical operator to get our accounts.
  • Deployment of TDDFT application
    • We confirmed if resource meets our requirements.
      • Installation and configuration of Globus, Intel compiler, Ninf-G
    • We uploaded the source code to each site and compile it.
  • Test
    • We did a Globus level test and a Ninf-G level test.
    • We simply ran TDDFT on 2 sites in short time.
  • Real run
    • We repeated to run TDDFT as many as we could.
    • We ran TDDFT as long as possible.

9
Trust and Access
  • Problems:
    • Different DN format requirements (access)
    • Different security policies (HTTP, monitor)
    • Different security software (GSISSH, access)
  • Need:
    • Clear/detail agreements
    • Procedure to resolve differences
  • PRAGMA 7 action item
10
Information Sharing
  • Problems:
    • Diverse administrative domains
    • Different operation schedules
    • Lack of info and documentations
  • Need:
    • Users and administrators guide
    • Procedures and schedules for maintenance
    • Bulletin-board/email notifications
  • PRAGMA 7 action item
11
Resources Requirements
  • Problems:
    • Inadequate network bandwidth
    • Inadequate memory capacity
    • Unforeseen software requirement details (versions, flavors, front-end/compute-notes)
  • Need:
    • Network/memory requirements
    • Detail software requirements
    • Procedure to update requirements
12
Resource Readiness
  • Problems:
    • Manually test each system
    • Resources and requirements change (out of synch)
  • Need:
    • Standard test suites run regularly
    • Automatically gather and publish sites info/status
  • Solutions in the Works:
    • Grid-status-tests, http://grid.ncsa.uiuc.edu/test/grid-status-test/
    • INCA, http://tech.teragrid.org/inca/www/documentation.html
13
Authentication Setup
  • Problems:
    • Manually setup/test user account on each site
    • Different security policies/setups on each site
  • Need:
    • Common authentication database
    • Automate setup/test processes
  • Solutions in the Works:
    • Rpm, tarball, MyProxy
  • Challenges:
    • Balance automation and local admin control
    • Manage/update CRLs
14
Application Setup
  • Problems:
    • Manually setup/test user environment and applications on each system
    • Differences among systems (software, path, version)
  • Need:
    • Provide common user environment setup
    • Automate setup replication
  • Solutions in the Works:
    • Gfarm, http://datafarm.apgrid.org
15
Resources/Jobs Match
  • Problems:
    • Incompatibility between some resources and jobs
      • Hardware capacity (network, memory, storage)
      • Software availability, version, configuration
      • System configuration (queue, PE)
  • Need:
    • Middleware wraps some differences
    • Help select right resources for users
  • Challenges:
    • Common application requirements
    • Provide choices for users
    • Match thru test runs
16
Job Submission
  • Problems:
    • Manually test/implement job submission for each system
    • Each system is different
      • job schedulers
      • batch queues/PEs setups
      • network/access setups
  • Need:
    • Easy user interface
    • Automate job submissions
  • Solutions in the Works:
    • NPACI GridPort, https://gridport.npaci.edu
17
Resources/Jobs Accounting
  • Problems:
    • Usage data available only on each system/node
  • Need:
    • Grid-enabled accounting
    • Usages per user, per job, in real time
  • Solutions in the Works:
    • KISTI
    • GridBank, http://www.gridbus.org
18
Resources Monitoring
  • Problems:
    • Different monitoring software don't work togather for centralized monitoring
      • SCMSWeb
      • Ganglia
  • Need:
    • Grid-enabled centralized monitoring
      • Standardazing Monitoring software
      • Build bridge-ware
  • Consideration:
    • GOC and NOC (APAN) collaboration
19
Jobs Monitoring
  • Problems:
    • Login/qstat on each system for job status
    • Miss real time job status change
  • Need:
    • Grid-enabled job monitoring
    • Notify users job status changes in real time
20
Fault Tolerance
  • Problems:
    • Not sure about problem cause
    • Doesn't recover from a fault
    • Manual clean-up
  • Need:
    • Middleware to provide for application developers
      • Good error reporting
      • Fault detection and recovery
      • Dynamically add/remove sites
      • Automatic clean-up after termination
  • Solutions in the Works:
    • Ninf-G, http://ninf.apgrid.org
