Work with PRAGMA Grid VOMS

 

 

1.      Start with the steps in PRAGMA Grid user guide to request resources and access, submit all required information specified in the user guide.

2.      Then the PRAGMA Grid administrator will make sure that your user certificate root CA files are installed in PRAGMA Grid VOMS trust database, then send you the following info/files/urls:

a.       VO name is PRAGMA-GRID-VO

b.      VO server name is rocks-83.sdsc.edu

c.       VO server certificate file

d.      PRAGMA root CA certificate files

e.       VO registration and configuration URL

f.        Vomses file

g.       URLs to adminstrators and users guide (This page)

3.      Application driver performs VO registration with a browser

a.       Import PRAGMA root CA certificate file as a root level certificate in your browser

b.      Import your own root CA certificate file as a root level certificate in the browser

c.       Import your personal certificate file (normally can be found in ~/.globus/usercert.p12) as a personal certificate in the browser

d.      Point the browser to the VO registration URL and register as a new user

4.      Application driver or his/her site administrator installs VOMS software on their own system

a.       Download a current version of VOMS client software from http://glite.web.cern.ch/glite/packages/

b.      Install VOMS client (installation instructions are in the INSTALL file)

c.       Make sure that the PRAGMA root CA certificate files are in your CA trust database

d.      Deposit the vomses file given in 3.f to $VOMS_LOCATION/etc/vomses. If installed by root, the directory and files should all be set to own by root:root and with permission 755 and 644 respectively

e.       Deposit the server certificate file given in 3.c in /etc/grid-security/vomsdir. If installed by root, the directory and files should all be set to own by root:root and with permission 755 and 644 respectively

5.      Application driver test access

a.       Make sure that your personal certificate files are in ~/.globus directory

b.      Set environmental variables for

                                                   i.      VOMS_LOCATION, point to VOMS installation directory

                                                 ii.      VOMS_USERCONF, point to the location of vomses files (see 4.d, or if user makes his/her own vomses files somewhere)

                                                iii.      Add $VOMS_LOCATION/bin in PATH

c.       Test access to the VO server and resources by running

                                                   i.      voms-proxy-init -debug -voms PRAGMA-GRID-VO

                                                 ii.      voms-proxy-info

                                                iii.      voms-proxy-list -voms PRAGMA-GRID-VO
