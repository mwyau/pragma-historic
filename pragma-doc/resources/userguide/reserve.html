Cross-site Reservation

Some special events may require dedicated resources among certain sites in PRAGMA testbed. Before we have a meta-scheduler to handle reservations automatically, we will try to follow a tentative manual procedure as below.

To keep things simple, we will dedicate resources at cluster level - which means that if a site decided to allow a cluster be used in a dedicated run, all nodes in the cluster should be made available to the dedicated run during the scheduled time. As how to accomplish this, each site is free to implement their own way. For those sites who like to know what's been done and worked, AIST has used a very simple and effective way on their clusters - make an alternate passwd file to include only the dedicated user (of course, all system accounts) and switch the passwd file during the dedicated time.

Here are the steps:

  1. Testing for access and application requirements on candidate clusters should be done before the dedicated run time
  2. Requester sends a request to PRAGMA Grid administrator
    • Spedify the purpose and the application for the dedicated run
    • Specify the desired dates of the reservation
    • Must name each cluster you wish to be reserved
    • The request must be made at least 10 days before the reservation starting date
    • Minor change to the request must be made at least 48 hours before the reservation starting date
    • Any major change to the request will need to go through the approval process as a brand new request
  3. The PRAGMA Grid administrator forwards the request to the named sites administrators within 24 hours.
  4. Each site will decide on approve or reject, and should notify the PRAGMA Grid administrator of their decision at least 5 days before the reservation starting date.
  5. The PRAGMA Grid administrator then inform the requester about the response from the sites and fix a schedule.
  6. The PRAGMA Grid administrator then email to pragma-grid-admin list, inform all application drivers and site administrators the scheduled duration, reserved sites and the application driver for the dedicated run
  7. One or two days before the reservation time, the PRAGMA Grid administrator will remind all application drivers and site administrators about this resource dedication.
  8. When the dedication time comes, the administrators of the dedicated sites should make the resources available for the dedicated user
  9. When the reservation period is over, or when the dedicated run is done before the dedicated ending time, the driver of the dedicated run should inform pragma-grid-admin list that the dedicated use is over.
  10. The administrators of the dedicated sites should open-up these dedicated resources to all PRAGMA testbed users again.
