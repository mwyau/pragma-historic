April 28, 2005
Kevin Dong

  1. About this document
    Computer Network Information Center (CNIC) is contributing a cluster system to the PRAGMA Testbed. This document is a users' guide for CNIC Grid Testbed and includes
    1. requirements for users
    2. procedures for getting an account
    3. hardware specifications and available Grid software
    4. resource utilization policy
    5. support staff available and mailing lists provided to users
    According to future administrative decisions, CNIC may modify this document and distribute the revised version through its web site.

  2. Requirements for users
    CNIC requests all users to fulfill the following conditions.
    1. All users must belong to one of member organizations of either PRAGMA, CNIC or SDB.
    2. All users participating in SDB must comply with the operating principles of CNIC.
    3. All users participating in PRAGMA must comply with the operating principles of PRAGMA.
    4. All users must have concrete purposes for using CNIC resources.
    5. All users must contribute to CNIC/PRAGMA through the use of CNIC/PRAGMA resources.
    6. All users must have a user certificate issued by a PRAGMA trusted CA.
    PRAGMA trusts the following CAs: AIST, AusGrid, BMG, DOEGrids, ESnet, KISTI, NCSA, NPACI, ThaiGrid, USM.
    Procedures for obtaining a user certificate are available on the PRAGMA-GOC home page.

  3. How to obtain an account
    In order to obtain an account for using CNIC resources, send the following information to Kevin Dong.
    1. Full Name
    2. Affiliation
    3. Address
    4. Country
    5. Phone number
    6. Email address
    7. Purpose for using CNIC Grid Testbed
    8. Desired account name (1st: 2nd : 3rd choices:)
    9. Shell (bash, csh, tcsh, etc.)
    10. ssh public key * **
    11. User certificate issued by a PRAGMA Trusted CA. ** ***
    The application will be reviewed by the administrators of CNIC resources within a few days of receipt of the application. The administrators will verify whether the applicant fulfills the requirements described in Section 2. If the application is approved, CNIC will create an account and send the necessary information to the applicant. Otherwise, the applicant will be notified that the request is rejected.

    * You can login to CNIC Grid Testbed via an ssh command using RSA authentication. CNIC will NOT inform you of your initial UNIX password.
    ** An ssh public key and your user certificate should be sent as attachments to your application.
    *** A mapping of your subject name to your local account will be added to the grid-mapfile on the CNIC Grid Testbed.

  4. Resource information
    CNIC is providing a Linux cluster, called CNIC Grid Testbed with the hostname pragma.sdg.ac.cn.
    CNIC Grid Testbed is a 8-nodes Linux Cluster whose interconnection is Quodrix Network. Each node has 4 IaitumnII 1.3GHz and 8GB memory. Users' home directories are shared among all nodes by NFS.

  5. Resource utilization policy
    1. Resource Reservation
      CNIC Grid Testbed is intended to be used for development and testing of Grid middleware and Grid applications in PRAGMA.

      You must apply exclusive use at least five business days before you intend to use CNIC Grid Testbed. Please send your application to grid-admin. The application should indicate

      • Your name
      • Your account
      • Number of CPUs/nodes which you would like to occupy
      • the start time / end time (time zone must be indicated)
        (you are allowed to occupy, at most, 24 hours for benchmarking)
      • the purpose of the experiments

      Your application will be reviewed by the administrators and you will be notified of the decision within a few business days of your application.
      CNIC will announce the schedule of the reservation via the users mailing list (grid-users@grid.cn).
      CNIC will provide the dedicated resources during the specified (or permitted) period. Other users will not be able to use (login) the dedicated resources during the period.

    2. Installation of additional software
      CNIC Grid Testbed can be used for R&D of Grid middleware and applications. You can install, test and evaluate your software on CNIC Grid Testbed. Since CNIC Grid Testbed are shared by many users, you are requested to test and debug your software thoroughly before installing it on CNIC Grid Testbed so that it will not damage the systems. If you need supervisor privilege for installing your software, please contact the support staff.

  6. Support staff and mailing lists

    CNIC Grid Testbed is maintained by support staff. Your requests will be processed in a best-effort manner by the staff. CNIC provides the following mailing list for users:

    • grid-users@grid.cn
      This mailing list is used mainly for announcements from administrators and support staff. All users are automatically included in the list.

    • grid-admin@grid.cn
      System administrators of CNIC Grid Testbed are on this mailing list. This mailing list is used for correspondence from all users. Any requests, questions, and comments should be sent to this mailing list.
